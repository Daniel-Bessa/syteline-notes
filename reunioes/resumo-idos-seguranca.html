Reunião com Alberto Gómez Martínez · Participantes: João Quelhas, Daniel Santos
Resumo
🗄️ O que é uma IDO ▶
•IDO = Intelligent Data Object — uma vista sobre várias tabelas SQL do SyteLine.
•A IDO standard não pode ser modificada diretamente — para alterar, cria-se uma extensão.
🔗 Tipos de Extensão de IDO ▶
•Extensão simples: cópia da IDO original à qual se adicionam propriedades/tabelas. Os forms têm de ser atualizados manualmente para usar a nova IDO.
•Extend and Replace (marcar check "Replace"): a nova IDO substitui a original em todos os forms do sistema automaticamente.
📌 Exemplo real: campo de descrição de Items demasiado curto (40/60 chars).
→ Criou-se propriedade extra numa tabela UD (User Extension Table) ligada a Items.
→ "Extend and Replace" na IDO SL Items → todos os forms passaram a usar o campo novo sem nenhuma alteração adicional.
Como criar um Extend & Replace:
1Abrir IDO maintenance → New IDO.
2Preencher "Extension of": nome da IDO original (ex: SL Items).
3Marcar o check Replace.
4Clicar Finish — a IDO é gerada com as propriedades herdadas + novas.
🖥️ Tipos de Form (ao criar de raiz) ▶
•Multiview: grid em cima + detalhe da linha selecionada em baixo. É o formato mais comum no SyteLine.
•Grid Only: todo o ecrã é uma grelha de dados, sem painel de detalhe.
•Detail Only: mostra só um registo de cada vez. Típico para forms de report ou registo de consumos.
🔐 Segurança — 3 Níveis ▶
Nível 1 — Infor OS (plataforma) Controla o acesso à plataforma e às aplicações ligadas (SyteLine, IDM, Factory Track, etc.).
Sem acesso aqui, não se entra no SyteLine.
Nível 2 — SyteLine (grupos) Grupos com autorizações: System Administration, Purchasing, etc.
Cada grupo define acesso a forms, IDOs, IDE e file server.
Ver em: Users > Group Authorizations
Nível 3 — Utilizador individual Permissões específicas do utilizador (adiciona ou restringe em relação ao grupo).
Editing Permission: controla se pode editar ou apenas visualizar.
⚖️ Regra: o mais restritivo prevalece.
Herança: Utilizador → Grupo → Site default.
Um utilizador pode herdar permissões do grupo + ter as suas próprias adicionais.
⚡ Application Events (triggers do SyteLine) ▶
•Funcionam como triggers SQL, mas a nível da aplicação SyteLine.
•Podem estar ligados a uma tabela/IDO específica ou a qualquer tabela do sistema.
•Podem disparar: notificações, emails, BOD (Business Object Documents via Ion), métodos.
Tipos mais usados:
IDO On Item InsertDisparado durante o insert — antes de gravar.
IDO Post Item InsertDisparado após o insert estar concluído.
IDO On Item DeleteDisparado durante o delete.
IDO On Item UpdateDisparado durante a atualização de um registo.
📌 Exemplo real: evento "PDF by Email" — quando uma tarefa de impressão termina, envia email com o documento gerado em anexo com parâmetros definidos.
🌐 Ion / Intelligent Connector ▶
•Ion: plataforma de integração da Infor — conecta SyteLine com sistemas externos.
•Intelligent Connector: monitoriza uma tabela SQL externa e, quando há alteração, atualiza o SyteLine automaticamente (quase em tempo real).
📌 Caso prático (cliente Citron):
Empresa envia XML para uma pasta → Ion apanha → extrai dados → preenche tabelas do SyteLine. Tudo automatizado sem intervenção manual.
