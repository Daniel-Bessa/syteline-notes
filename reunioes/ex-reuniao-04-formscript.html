EX 1 Explorar o FormScript de Purchase Orders
🎯 Objetivo: Navegar pelo FormScript do form Purchase Orders e perceber a estrutura de eventos e funções.
Passos
Abrir o form Purchase Orders e entrar no Designer (lápis).
Ir à secção FormScript (dentro do designer).
Usar o dropdown para ver a lista de todas as funções/sub-rotinas do FormScript.
Localizar a função DefaultVendCurCode (ou similar) — verificar que é uma Function As Integer (devolve inteiro).
Ir ao campo VendNam na grid → ver os Events associados.
Em Event Handlers, localizar o evento "Data Change PO VenNam".
Ver a sequência de ações que este evento executa (Form Script Method, Method Call, Generate Event, etc.).
💡 A sequência é executada de cima para baixo, por ordem de Sequence Number. Cada passo pode ser um FormScript Method, Method Call, Generate Event ou RunScript.
EX 2 Explorar os Eventos Standard do form Purchase Orders
🎯 Objetivo: Verificar os eventos standard disponíveis e perceber a diferença entre "Form Copy" e "Form Copy Complete".
Passos
No designer do form Purchase Orders, ir à secção Event Handlers.
Filtrar por "Standard" para ver os eventos standard disponíveis.
Localizar Standard Form Copy — verificar quando é disparado (ao clicar em Copiar).
Localizar Standard Form Copy Complete — verificar que é o evento de conclusão da cópia.
Verificar se existe Standard Form Predisplay Complete — não deve existir (este não tem par de "Complete").
Localizar Standard Form Save e Standard Object Save — notar que são dois momentos distintos.
EX 3 Impedir delete permanente via Standard Operations
🎯 Objetivo: Desativar o botão de delete permanentemente na cópia de trabalho UE_DS_Items (Opção A — nunca pode apagar).
Passos
Abrir o form UE_DS_Items (cópia criada no exercício anterior) e entrar no Designer.
Selecionar a coleção principal do form (normalmente "Items").
No painel direito, ir à secção Standard Operations.
Desmarcar (desativar) a opção Delete.
Guardar as alterações.
Abrir o form e confirmar que o botão de delete está cinzento/inativo.
EX 4 Escrever uma função FormScript que para a execução (return -1)
🎯 Objetivo: Criar no FormScript da cópia de trabalho uma função que devolve -1, bloqueando qualquer ação que a chame. Base para validações condicionais.
Código a implementar
Function StopAction() As Integer Return -1 End Function
Passos
Abrir o form UE_DS_Items no Designer.
Ir ao FormScript.
No final do código existente (antes do fim da classe), adicionar a função StopAction como acima.Atenção: a função tem de estar dentro da classe do FormScript, não fora.
Guardar o FormScript.
Verificar no dropdown de funções que StopAction aparece listada.
Criar um evento de teste (ex: Standard Form Copy) com um passo "Form Script Method" que chama StopAction.
Testar: tentar copiar um registo — a ação deve ser bloqueada silenciosamente (return -1 para a execução).
Remover o passo de teste do evento para não afetar o form.
💡 Regra: Function devolve 0 = sucesso, continua; devolve -1 = erro, para tudo.
Esta técnica é usada para: impedir delete condicional, validar campos antes de guardar, bloquear ações em determinados estados.
