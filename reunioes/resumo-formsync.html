Resumo
🏗️ Níveis de um Form (do mais genérico ao mais específico) ▶
User Versão personalizada de um utilizador individual. Carrega primeiro. Mais específico
Group Versão do grupo de utilizadores. ↑ 2º
Site Versão da empresa — é aqui que trabalhamos normalmente. ↑ 3º
Vendor Standard da Infor — nunca modificável. Último recurso. Padrão
💡 Para ver em que versão estás: "About This Form" dentro do designer.
Trabalhamos sempre sobre a versão Site.
💾 Backup — Exportar com FormSync (antes de editar) ▶
⚠️ Antes de editar qualquer form, fazer sempre um export como backup.
  1. 1Abrir o form FormSync.
  2. 2Definir Origem e Destino com o mesmo ambiente e site (ex: TRN→TRN ou PRD→PRD).
  3. 3Em Utilities: desmarcar tudo, selecionar apenas Forms.
  4. 4Filtrar pelo nome do form (ex: "Items").
  5. 5Ação: Export.
  6. 6Nomear o ficheiro: [Form]_[Ambiente]_[Data] → ex: Items_TRN_2024-06-20
  7. 7É gerado um ficheiro XML — é este o backup.
↩️ Reverter — Importar o Backup ▶
  1. 1Abrir FormSync → separador Import.
  2. 2Clicar em Upload e arrastar o XML exportado anteriormente.
  3. 3Clicar em Load → depois Import.
  4. 4Fazer Unload All Global Forms Object (em Utilities) para atualizar o form em memória.
💡 Podes também abrir o XML no Notepad++ e comparar dois ficheiros (antes e depois) para ver exatamente o que mudou.
🔒 Form Control — Gerir Bloqueios ▶
•Form Form Control: mostra quem tem um form bloqueado.
•Pesquisar o form → Refresh Status → aparece o utilizador que tem o lock.
•Para desbloquear: selecionar Unlock → Execute Action.
👤 Regra de cortesia: sempre pedir ao colega para desbloquear antes de entrar — evita conflitos de edição. Só usar o Form Control para desbloquear se o colega estiver de férias ou indisponível.
📋 Copiar um Form (para testes) ▶
  1. 1Abrir o designer do form base.
  2. 2Opção Copy Form.
  3. 3Nome: UE_[Iniciais]_[NomeForm] → ex: UE_DS_PurchaseOrders
  4. 4Atualizar o Caption (nome visível) para identificar a cópia.
✅ Alterações na cópia não afetam o form original. Usar para testes livres.
🔗 Extensões de Form ("Extend Form") ▶
•Em vez de copiar, cria-se uma extensão do form base.
•Opção "Replace Base Forms": quando ativa, a extensão substitui o original em todo o sistema.
•Útil quando o cliente não quer form novo mas quer adições ao existente.
•Trabalhar com extensões é mais trabalhoso — cada campo tem de ser ativado antes de editar.
🧠 Quando Criar Novo vs Copiar vs Extender ▶
•Copiar: form de destino é muito semelhante ao existente (mudar nome + ocultar/adicionar campos).
•Extender: cliente não quer form novo nem modificar o standard.
•Criar de raiz: normalmente para forms de consulta ou que chamam reports — não para forms com lógica complexa.
📌 A decisão vem sempre da análise do consultor (gap analysis) — o developer implementa o que lhe é pedido.
